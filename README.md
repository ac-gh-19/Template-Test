# Template-Test

A basic TypeScript template with npm packages installed for static typing, testing with Jest, and code formatting with Prettier.

## Features

- **TypeScript**: Static typing for better code quality and developer experience
- **Jest**: Testing framework with TypeScript support
- **Prettier**: Code formatter for consistent code style

## Prerequisites

- Node.js (v16 or higher recommended)
- npm (comes with Node.js)

## Installation

Install all dependencies:

```bash
npm install
```

## Project Structure

```
.
├── src/
│   ├── index.ts       # Main source file
│   └── index.test.ts  # Test file
├── dist/              # Compiled JavaScript output (generated by build)
├── .prettierrc        # Prettier configuration
├── jest.config.js     # Jest configuration
├── tsconfig.json      # TypeScript configuration
└── package.json       # Project dependencies and scripts
```

## Available Scripts

### Development

- `npm run dev` - Run the TypeScript code directly using ts-node
- `npm run build` - Compile TypeScript to JavaScript in the `dist/` directory
- `npm start` - Run the compiled JavaScript from `dist/`

### Testing

- `npm test` - Run all tests with Jest
- `npm run test:watch` - Run tests in watch mode
- `npm run test:coverage` - Run tests with coverage report

### Code Formatting

- `npm run format` - Format all TypeScript files in `src/` with Prettier
- `npm run format:check` - Check if files are formatted correctly
- `npm run lint` - Check code formatting (same as format:check)

## Usage Example

The template includes a simple example with math functions. To run it:

```bash
# Run with TypeScript directly
npm run dev

# Or build and run
npm run build
npm start
```

## Writing Tests

Tests are written using Jest and should be placed in the `src/` directory with the `.test.ts` extension. Example:

```typescript
import { add } from './index';

describe('add function', () => {
  it('should add two numbers correctly', () => {
    expect(add(2, 3)).toBe(5);
  });
});
```

Run tests with:

```bash
npm test
```

## Configuration Files

### TypeScript (tsconfig.json)

Configured for:
- Target: ES2020
- Module: CommonJS
- Strict type checking enabled
- Source files in `src/`, output to `dist/`

### Jest (jest.config.js)

Configured for:
- TypeScript support via ts-jest
- Node.js test environment
- Test files: `*.test.ts` or `*.spec.ts`
- Coverage collection from `src/` directory

### Prettier (.prettierrc)

Configured for:
- Semicolons enabled
- Single quotes
- 2-space indentation
- 80 character line width
